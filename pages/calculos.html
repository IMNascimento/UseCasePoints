TAP

Simples: Um ator é considerado simples se ele representa outro sistema com uma API definida.

Médio: Um ator é considerado médio se é uma interação com outro sistema através de um protocolo ou se é uma interação humana através de linha de comando.

Complexo: Um ator é considerado complexo se ele interage com o sistema através de uma interface gráfica.

OBS: Passe os totais de atores simples,medios e complexo.

TUCP

Existe 3 tipos de casos de uso e cada um deles é definido pelo número de transações que ocorre nele uma transação geralmente representa qualquer alteração em um banco de dados sendo que:

Um caso simples possui até 3 transações.

Um caso medio possui até 4 e 7 transações

Um caso complexo possui mais de 7 transações

Fatores Técnico

Sistema distribuído: é uma coleção de programas de computador que utilizam recursos computacionais em vários pontos centrais de computação diferentes para atingir um objetivo comum e compartilhado.

Tempo de resposta: é a métrica que mede o tempo em que uma requisição leva para responder.

Eficiência(on-line): execução com alta perfomace com poucas linhas de codigo

Processamento interno complexo:é um conjunto de partes ou subsistemas com processamentos internos singulares, conectadas entre si

Código deve ser reutilizável: reutilização de código, é o uso de software existente, ou do conhecimento de software, para a construção de um novo software.

Facilidade de instalação: Conhecimeto da ferrameta nessesaria para processo de intalação

Usabilidade: é a habilidade de um sistema em conduzir um usuário a concluir seus objetivos com eficácia, eficiência e satisfação em uma experiência de produto, serviço ou interface.

Portabilidade: funciona como uma compra de dívida. Ao levar seu empréstimo para um outro banco, ele paga seu contrato com a antiga instituição e faz um novo.

Facilidade de manutenção: é o resultado de testes realizados no protótipo ou virtualmente em modelos 3D de um produto, com o objetivo de avaliar a facilidade de acesso e intervenção em peças sujeitas a manutenção ou reparo frequentes.

Acessos simultâneos (concorrência): consideramos usuários simultâneos aqueles que estão executando a mesma ação ao mesmo tempo.

Aspectos especiais de segurança: as inspeções de segurança são procedimentos realizados com o intuito de detectar situações perigosas.

Acesso direto para terceiros: o cliente final possa acessar o mercado diretamente sem passar por uma infraestrutura tecnológica.

Facilidades especiais de treinamento: conjunto de procedimentos, que tem o objetivo de contribuir para a evolução de um colaborador a curto e longo prazo.

Pontuação será entre 0 e 5:

  • 0 é ruim ou péssimo.
  • 3 é razoável
  • 5 é ótimo ou bom
